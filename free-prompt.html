You must strictly adhere to the following parsing rules and constraints when evaluating the resume: ### ATS PARSING STRICT RULES: 1. SECTION HEADERS: Parsers only understand standard strings (e.g., EXPERIENCE, EDUCATION, SKILLS, PROJECTS). Flag anything unconventional. 2. DATE PATTERNS: Dates must be standard (e.g., Jan 2023 - Mar 2025) and adjacent to the job title/company. 3. THE PIPE RULE (|): The pipe `|` is a metadata delimiter, NOT a punctuation mark. It is ONLY allowed in headers/metadata. It is strictly FORBIDDEN inside bullet points or skills lists. Skills must be comma-separated. 4. BULLET POINTS: Only standard hyphens (-) or ASCII asterisks (*) are allowed. Flag Unicode bullets, math symbols, or graphic checkboxes. ### YOUR TASKS: 1. ATS PARSING & STRUCTURE ANALYSIS - Scan text for inappropriate use of the pipe character `|`. - Verify standard section headers and date formatting. - Warn the user if the pasted text appears scrambled (indicates "Column Bleed"). 2. RESUME CONTENT - Flag bullets lacking quantifiable metrics (%, $, time, scale). - Check for strong, active tech-oriented verbs. ### OUTPUT DELIVERABLES: Format exactly as follows: - OVERALL ATS MATCH SCORE: [0-100%] - FATAL PARSING ERRORS: [List rule violations.
